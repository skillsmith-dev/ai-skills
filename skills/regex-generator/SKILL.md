# 正则表达式生成器

> 用户说出「正则表达式」「regex」「正则匹配」等关键词时激活。

## 输出模板

```
## 🔧 正则表达式方案

### 📝 你的需求
{用自然语言复述用户的匹配需求}

### ✅ 推荐正则
```
{正则表达式}
```

**在线测试**: regex101.com（粘贴即可测试）

### 📖 逐段解释
| 部分 | 含义 |
|------|------|
| {部分1} | {解释} |
| {部分2} | {解释} |

### 🧪 测试用例
| 输入 | 是否匹配 | 匹配结果 |
|------|---------|---------|
| {匹配示例1} | ✅ | {匹配内容} |
| {不匹配示例1} | ❌ | - |
| {边界示例1} | ⚠️ | {说明} |

### 🔄 常见变体
- **更严格**: {更精确的正则}
- **更宽松**: {容错更多的正则}
- **不同语言**: {Python/JS/Java语法差异}
```

## 执行流程

### Step 1：理解需求
用户描述要匹配什么内容，如：
- "匹配中国大陆手机号"
- "提取HTML中的所有链接"
- "验证密码强度：8位以上含大小写数字"

### Step 2：生成正则
考虑：
- 边界情况（空串、特殊字符、Unicode）
- 性能（避免灾难性回溯）
- 可读性（复杂正则加注释）

### Step 3：验证
用代码验证正则的正确性

### Step 4：输出

## 质量红线
1. **必须逐段解释** — 正则不解释等于没用
2. **测试用例必须包含匹配和不匹配** — 证明正则正确
3. **标注边界情况** — 如手机号170/191等新号段
4. **给出语言差异** — Python和JS正则有区别
